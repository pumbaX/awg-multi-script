## Быстрый старт

```bash
sudo curl -fsSL https://raw.githubusercontent.com/pumbaX/awg-multi-script/main/awg2.sh -o /usr/local/bin/awg2 && sudo chmod +x /usr/local/bin/awg2 && sudo awg2
```

Запуск в любой момент:
```bash
sudo awg2
```

## Уровни обфускации

При создании сервера (пункт 2) скрипт спрашивает уровень:

| Уровень | Параметры | Совместимость | DPI стойкость |
|---|---|---|---|
| **1. Базовый** | H ranges + S1-S4 + Jc junk | ✅ все клиенты | Высокая |
| **2. +I1** | + один сигнатурный пакет (QUIC/WebRTC/SIP/DNS) | ✅ современные | Очень высокая |
| **3. Полный CPS** | + I2-I5 энтропийные пакеты | ⚠️ может ломать старые клиенты | Максимум |

**По умолчанию — уровень 1.** Этого хватает в 95% случаев включая ТСПУ в РФ. Поднимай выше только если базовый блокируется.

## Профили мимикрии (для уровней 2 и 3)

```
Works (проверено на ТСПУ РФ):
1) ★ QUIC Full      — полноразмерный Chrome-like (~1200B, только I1)
2)   QUIC Mini      — ультракомпактный (~40B, I1-I5 с <r 16>)

Kinda (может работать):
3)   WebRTC Combined — STUN + DTLS 1.2 + RTCP
4)   SIP             — OPTIONS/REGISTER/TRYING (случайно)
5)   DNS Response    — как у оригинальной Amnezia
```

CPS-генератор (порт [payloadGen от Sketchystan1](https://github.com/Sketchystan1/payloadGen)) собирает байт-валидные пакеты на Python:

- **QUIC Full** — полноразмерный QUIC Initial ~1200B с правильным varint Payload Length. First byte `0xc1` или `0xc5` (reserved bits 0 или 1, PN_len=2) — точно как Chrome. **Не влезает в QR** — показывается только текст конфига.
- **QUIC Mini** — ультракомпактный QUIC Initial ~38-45B с `<r 16>` в конце I1. First byte `0xc0` или `0xc1`. Все I1-I5 заполнены.
- **WebRTC Combined** — STUN Binding Request с валидным FINGERPRINT (CRC32 XOR 0x5354554E) + DTLS 1.2 ClientHello с SNI + RTCP tail. I1 + 4 mini QUIC.
- **SIP** — случайный выбор OPTIONS/REGISTER/TRYING с полными заголовками (Allow, Supported, Accept-Language, User-Agent: Linphone/Zoiper/MicroSIP/Asterisk).
- **DNS Response** — DNS Response с `<r 2>` TXID. Как у оригинальной Amnezia.

**Padding опция** — для QUIC Full и WebRTC спрашивает `Padding до 1200B (реалистичнее)? [y/N]`. Через env `CPS_PAD=1`.

**Domain pool из payloadGen** — 68 доменов микс RU+WORLD (google.com, yandex.ru, vk.com, mail.ru, gosuslugi.ru, sberbank.ru, cloudflare.com, github.com...).

> Все I-пакеты начинаются с тега `<b 0x...>` — критическое требование парсера amneziawg-go.
> `<r>` режется на куски ≤999 байт для совместимости со старыми клиентами.

## Параметры обфускации AWG 2.0 (по официальному мануалу)

Скрипт генерирует параметры строго по [официальному мануалу AmneziaWG](https://docs.amnezia.org/documentation/amnezia-wg/):

| Параметр | Лимит мануала | Наш диапазон | Описание |
|---|---|---|---|
| **Jc** | 1 ≤ Jc ≤ 128 | 4-12 | Количество junk-пакетов перед handshake |
| **Jmin** | Jmin < Jmax | 3-15 | Мин. размер junk-пакета |
| **Jmax** | Jmin < Jmax ≤ 1280 | 50-120 | Макс. размер junk-пакета |
| **S1** | S1 ≤ 1132 | 15-150 | Padding Init-пакета |
| **S2** | S2 ≤ 1188 | 15-150 | Padding Response-пакета |
| **S3** | AWG 2.0 ext | 8-55 | Padding Cookie-пакета |
| **S4** | AWG 2.0 ext | 4-27 | Padding Data-пакета (≤32) |
| **H1-H4** | 5 ≤ H ≤ 2^31-1 | см. ниже | Magic numbers — диапазоны |

**Обязательно:** `S1 + 56 ≠ S2` — проверяется автоматически.

### H1-H4 sub-квадранты

Мануал AmneziaWG: `H1-H4 must be unique, recommended 5 ≤ H ≤ 2147483647`.

Скрипт разделяет recommended диапазон на 4 подсегмента (~2^29 каждый):

```
H1 ∈ Sub-Q1 [5          .. 536,870,911]       (до 2^29-1)
H2 ∈ Sub-Q2 [536,870,912 .. 1,073,741,823]    (2^29 .. 2^30-1)
H3 ∈ Sub-Q3 [1,073,741,824 .. 1,610,612,735]  (2^30 .. 3*2^29-1)
H4 ∈ Sub-Q4 [1,610,612,736 .. 2,147,483,647]  (3*2^29 .. 2^31-1)
```

**Преимущества:**
- Все значения в recommended диапазоне мануала
- Windows AmneziaVPN UI принимает значения (UI баг с 2^31-1 обойдён)
- H1-H4 гарантированно уникальны и не пересекаются
- Ширина каждого диапазона 500M-900M — максимальная энтропия

Проверить конфиг можно через [AWG Analyzer](https://pumbax.github.io/awg-analyzer/) — идеал **12/12 баллов** для H1-H4.

## Управление клиентами (пункт 3)

```
⚙  Управление клиентами
  1) Добавить клиента
  2) Переименовать клиента
  3) Удалить клиента
  4) Показать QR клиента
  5) Показать конфиг клиента (текст)
  0) Назад в главное меню
```

**Добавление** — генерит ключи, выделяет IP из подсети, предлагает профиль мимикрии, записывает в серверный конфиг и создаёт `/root/<имя>_awg2.conf`. Использует `awg syncconf` — другие клиенты не теряют соединение.

**Переименование** — показывает список с именами и VPN IP, запрашивает новое имя, обновляет `# comment` в серверном конфиге, переименовывает файл.

**Удаление** — список клиентов, подтверждение, удаление из runtime (`awg set awg0 peer <pk> remove`), вырезание блока `[Peer]` из конфига, удаление файла. Перед любым изменением — автоматический бекап.

**QR-код (пункт 4)** — вывод QR в терминал через qrencode. Автоматически проверяет размер: конфиги до 2800 байт → QR, выше → предложение показать текстом.

**Показать конфиг (пункт 5)** — выводит полный текст конфига для копирования или `scp`.

## Сбросить сервер (пункт 11)

Чистая переустановка сервера **без удаления пакетов**. Идеально для пересоздания с новыми параметрами (сменить регион, обновить обфускацию) без полной переустановки.

**Workflow:**
```
1. Подтверждение: "Подтверди сброс [yes/N]"
2. Предложение бекапа: "Создать бекап перед сбросом? [Y/n]"
   → Y — сохраняет текущий конфиг в ~/awg_backup/
3. awg-quick down — останавливает интерфейс
4. Удаляет iptables правила NAT/FORWARD
5. Удаляет серверный конфиг + все .bak.*, .pre_rename.*, .pre_delete.*
6. Удаляет клиентские конфиги /root/*_awg2.conf
7. Удаляет UFW правила AmneziaWG
8. Сбрасывает SERVER_REGION к дефолту
```

**НЕ трогает:** пакеты amneziawg / amneziawg-tools, kernel module, бинарники, лог `/var/log/awg-Toolza.log`, бекапы в `~/awg_backup/`.

После сброса сразу можно пункт 2 — создать новый сервер с нуля.

### Разница между пунктами 10, 11, 12

| Пункт | Что удаляет | Что оставляет | Когда использовать |
|---|---|---|---|
| **10. Очистить клиентов** | Все `[Peer]` блоки + файлы клиентов | Сервер работает, можно добавлять новых | Хочу стереть всех клиентов, но сервер оставить |
| **11. Сбросить сервер** | Весь конфиг + клиенты + firewall | Пакеты, бинарники, бекапы | Хочу пересоздать сервер с нуля без переустановки |
| **12. Удалить всё** | Всё выше + пакеты + PPA | Ничего | Хочу удалить AmneziaWG полностью |

## DPI тест (пункт 7)

Захватывает пакеты от выбранного клиента через tcpdump и анализирует:

| Тип | Что детектит |
|---|---|
| **SIP** | REGISTER, INVITE, OPTIONS, BYE, CANCEL, ACK |
| **TLS** | ClientHello, ServerHello, CKE, ChangeCipherSpec, Application Data |
| **QUIC** | Long Header (Initial/0-RTT/Handshake) + Short Header (1-RTT) |
| **DTLS** | record + ClientHello |
| **DNS** | wire-format query |
| **AWG data** | обфусцированные пакеты с кастомными H-заголовками |

Три возможных результата:
```
✓ DPI тест пройден — CPS chain из 3 пакетов
✓ Обфускация работает (CPS уже прошли)
○ Ничего не поймали — попробуй переподключиться
```

## Подводные камни

> ⚠️ **Endpoint = IP, не домен.** Доменное имя в `Endpoint = ` вызывает дедлок переподключения на Keenetic и других роутерах.

> ⚠️ **MTU должен совпадать** между сервером и клиентом. По умолчанию 1380, для AWG 2.0 + CPS лучше 1320.

> ⚠️ **I-параметры должны совпадать** между сервером и клиентом. После пересоздания сервера обязательно переимпортируй конфиг на всех клиентах — иначе handshake висит без ошибок в dmesg.

> ⚠️ **AmneziaVPN < 4.8.12.7 не поддерживает AWG 2.0.** Обнови клиент до последней версии.

> ⚠️ **Параметры v5.2 и старше** могут иметь кривое распределение H1-H4. Работают, но для идеального `12/12` в анализаторе пересоздай сервер через v6.2.

> ⚠️ **LXC/OpenVZ контейнеры** — `sysctl net.ipv4.ip_forward=1` может требовать прав хоста. Скрипт пробует `/proc/sys/...` как fallback, но если не работает — включи IP forwarding на хосте виртуализации.
